## 背景

等待交互场景此前做了结构化按钮、选项和特殊样式渲染，但真实使用中存在两个问题：

1. 交互执行链复杂，容易出现前台焦点、TTY 写入、终端状态不一致带来的失败。
2. 过度结构化的 UI 容易误导用户，以为可以在面板内完整完成确认流程。

## 决策

等待交互场景统一降级为“纯文本提醒”：

1. 任务卡片：
   - 不再渲染交互按钮、选项按钮、确认按钮、特殊选择态样式。
   - 仅展示一条明显提醒和终端中的原始确认文案。
   - 用户需要前往终端手工处理。
2. pill：
   - 不再展示具体终端输出文案。
   - 统一展示固定提醒：`您的任务需要手工确认。`

## 实施

1. `TaskStrategySessionSignalParser`
   - `waitingInput` 时固定返回提醒文案。
2. `IslandView`
   - 任务卡片中 `waitingInput` 改为只读文本区；
   - 移除结构化交互按钮渲染分支；
   - 恢复点击整卡前往终端处理。
3. 测试
   - 增加 `waitingInput` pill 固定提醒测试；
   - 增加确认文案格式化测试。
