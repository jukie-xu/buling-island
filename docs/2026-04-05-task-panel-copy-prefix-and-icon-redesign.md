# 任务面板固定文案前缀与图标重排设计

## 背景

现有任务卡片虽然已经能展示 `session.title`、用户最后一次提问和任务结果，但正文仍然是两段裸文本。用户需要自行判断：

- 哪一行是会话标题
- 哪一行是用户任务
- 哪一行是任务状态
- 哪一段是等待确认时的操作说明
- 底部元信息和正文状态分别代表什么

这会让任务面板在多会话并存时可读性明显下降。

## 目标

为任务卡片建立固定展示 contract，让用户一眼分辨：

1. 当前卡片属于哪个工具/会话
2. 用户最后一次提交的任务是什么
3. 任务当前所处状态是什么
4. 是否需要用户介入
5. 该任务来自哪个终端与 tty

## 固定展示格式

每张任务卡片统一拆成 4 个语义区块：

1. 顶部标题区
- 左侧保留运行状态点
- 增加策略标识 badge，例如 `Codex` / `Claude`
- 使用 `会话` badge 明确标识标题含义
- 标题右侧保留 pin / mute 控件

2. 任务区
- 固定前缀：`任务`
- 固定图标：文本/对话类图标
- 内容：当前 session 缓存的最后一次真实用户提问

3. 状态区
- 固定前缀：`状态`
- 固定图标：根据 lifecycle 映射
- 内容：
  - `running`：最新有效输出或 `处理中…`
  - `waitingInput`：`等待手工确认`
  - `success`：`任务已完成`
  - `error`：错误摘要
  - `idle`：完成后仍显示 `任务已完成`，否则显示 `等待新任务`
  - `inactiveTool`：`未识别到任务工具`

4. 来源区
- 固定前缀：`来源`
- 固定图标：终端图标
- 内容：`策略名 · 终端类型 · tty`

等待确认时增加额外操作区：

- 固定前缀：`操作`
- 固定图标：手动操作类图标
- 内容：交互详情或审批命令

## 设计原则

- 不让用户猜测每一行的语义
- 所有状态文案固定收口，避免不同策略输出直接污染 UI
- 图标仅承担快速识别作用，核心信息仍由中文前缀明确表达
- 任务面板与数据层 contract 解耦，文案结构由 `TaskSessionTextToolkit` 统一生成

## 实现落点

- `TaskSessionTextToolkit.taskPanelPresentation(from:)`
  - 统一生成 `taskLine / statusLine / detailLine / lifecycleLabel`
- `TaskPanelView`
  - 负责 badge、图标、分区排版
  - 不再直接把 `secondaryText` 裸渲染到界面

## 预期结果

任务面板中的每张卡片都能快速回答下面 4 个问题：

- 这是哪个 session
- 用户让它做什么
- 它现在做到哪一步
- 要不要我介入
