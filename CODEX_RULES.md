# Codex 开发规则（强制）

本文件是当前项目的 Codex 开发强制规则。每次开发任务都必须严格执行，不能跳过。

## 1. 任务先设计，后开发（强制）

每次接到任务后，必须先将任务转化为一份完整设计方案，设计文档放在 `docs/` 目录下，然后才能进入代码开发。

设计文档最低要求：
- 背景与目标
- 范围（包含项 / 不包含项）
- 方案设计（模块、流程、数据结构或接口）
- 变更清单（涉及文件与改动点）
- 验证方案（如何确认功能正确）
- 风险与回滚方案

命名建议：
- `docs/YYYY-MM-DD-<task-name>-design.md`

## 2. 严格按设计实施（强制）

代码实现必须基于已落库的详细设计文档执行。

执行要求：
- 实现内容与设计保持一致
- 如果实现过程中方案变化，必须先更新 `docs/` 中对应设计文档，再继续改代码
- 禁止在没有设计文档或设计文档不完整的情况下直接改代码

## 3. 代码变更后必须本机部署更新（强制）

只要任务涉及任何代码变更（无论变更大小），在任务结束前必须执行：

```bash
./install-local.sh
```

用于完成本机 App 的部署与更新验证。未执行该脚本，任务不得视为完成。

## 4. 任务完成前检查清单（必过）

- [ ] 已在 `docs/` 下新增或更新完整设计文档
- [ ] 代码实现与设计文档一致（变更已同步到文档）
- [ ] 若有代码变更，已执行 `./install-local.sh`
